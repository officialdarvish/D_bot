# اصلاحات یکپارچه‌سازی MikroTik (D Bot ↔ پنل PPP Secret)

این نسخه باگ‌های اتصال بات تلگرام به **پنل MikroTik PPP Secret** (آدرس API:
`http://88.198.186.39:8090/api`) را برطرف می‌کند. بات فقط از طریق REST API پنل با
میکروتیک کار می‌کند (هیچ اتصال مستقیم RouterOS وجود ندارد) و آداپتور آن
`app/services/mikrotik_service.py` است.

قرارداد API پنل تأیید شد و کاملاً سالم است؛ همهٔ باگ‌ها سمت **بات** بودند.

## باگ‌های برطرف‌شده

1. **اکانت تست برای میکروتیک اصلاً کار نمی‌کرد** — `app/bot/handlers/public/test_account.py`
   - شرط وجود Inbound فقط برای `xui` اعمال می‌شود (میکروتیک Inbound ندارد).
   - شاخهٔ ساخت کاربر میکروتیک اضافه شد (`create_user_on_plan`) و رمز تولیدشده ذخیره می‌شود.
   - `server_type`، `password` و `service_id` به کارت ارسال می‌شوند.

2. **«باطل کردن و ارسال جدید» رمز جدید میکروتیک را گم می‌کرد** — `my_services.py:revoke_service`
   - قبلاً ارسال به وجود `sub_link` گره خورده بود (برای میکروتیک همیشه `None`) و کاربر
     پیام موفقیت می‌دید ولی رمز جدید را دریافت نمی‌کرد و دیگر نمی‌توانست وصل شود.
   - حالا برای میکروتیک، نام کاربری و **رمز جدید** مستقیم ارسال می‌شود.

3. **صفحهٔ جزئیات سرویس رمز میکروتیک را نشان نمی‌داد** — `my_services.py:service_detail_text`
   - برای میکروتیک به‌جای «لینک اتصال/Happ/V2rayNG»، نام کاربری + رمز عبور + راهنمای
     میکروتیک نمایش داده می‌شود. (callbacks: `render_detail`, `auto_refresh_service_page`, `revoke`)

4. **خریدِ ارز دیجیتال (NowPayments) رمز میکروتیک را ارسال نمی‌کرد** — `app/api/main.py`
   - پارامتر `password` برای سرورهای میکروتیک به کارت سرویس افزوده شد.

5. **خرید رایگانِ ادمین رمز/راهنمای میکروتیک را ارسال نمی‌کرد** — `app/bot/handlers/public/buy.py`
   - فراخوانی `send_service_info` با `server_type` و `password` کامل شد.
   - همچنین در فراخوانی تأیید ادمین، رمز فقط برای میکروتیک ارسال می‌شود (قبلاً UUID
     کاربر xui به‌اشتباه به‌عنوان «رمز عبور» نمایش داده می‌شد).

6. **تمدید، سکرتِ غیرفعال‌شده را دوباره فعال نمی‌کرد** — آداپتور + تمام مسیرهای تمدید
   - متد `set_enabled` / `enable_user` و کمکی `renew_user` (update سپس enable) به
     `mikrotik_service.py` اضافه شد.
   - مسیرهای تمدید (`buy.py`، `my_services.py` تغییر پلن و تمدید کیف‌پول، `reseller.py`)
     حالا از `renew_user` استفاده می‌کنند تا اگر پنل کاربر را به‌خاطر انقضا/اتمام حجم
     غیرفعال کرده باشد، پس از تمدید دوباره فعال شود.

7. **تشخیص «کاربر یافت نشد» شکننده بود** — `mikrotik_service.get_user`
   - علاوه بر `not found`، حالا `not exist` و `404` هم تشخیص داده می‌شوند تا پاک‌سازیِ
     رکوردِ حذف‌شده در پنل درست انجام شود.

8. **به‌روزرسانی دوره‌ای مصرف/انقضای میکروتیک وجود نداشت** — `app/jobs/server_sync.py` + `app/main.py`
   - Job جدید `sync_mikrotik_usage` هر ۵ دقیقه `used_bytes`/`total_bytes`/`is_active`/
     `expires_at` را از پنل می‌خواند تا هشدارهای حجم/انقضا و لیست سرویس‌ها داده‌های تازه
     داشته باشند. (اعمال محدودیت همچنان سمت پنل/میکروتیک است؛ این job فقط آینه می‌کند.)

9. **دام walrus** — `app/api/admin_web.py:server_edit`
   - `if server_type := getattr(...) == 'mikrotik'` به دو خط ایمن تبدیل شد.

## اعتبارسنجی
- `python -m compileall -q app` بدون خطا.
- آزمون زندهٔ کامل روی روتر **DE** پنل واقعی: ۴۰۴→None، create، disable،
  renew (patch+enable → `disabled=false`)، و delete همگی موفق.

## نکتهٔ پیکربندی
هنگام افزودن سرور در پنل ادمین بات، نوع سرور را **MikroTik** انتخاب کنید و:
- **Panel URL / API Base** = `http://88.198.186.39:8090` (آداپتور خودش `/api` را اضافه می‌کند)
- **Router name** = دقیقاً نام روتر در پنل (`Ned` یا `DE`)
- **Password / API token** = مقدار `api_key` از `config.json` پنل
